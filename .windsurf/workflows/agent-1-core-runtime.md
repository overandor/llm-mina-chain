# Agent 1 — Core Runtime

## Scope

**You own:**
- `rust/src/crypto.rs`
- `rust/src/storage.rs`
- `rust/src/network.rs`
- `rust/src/consensus.rs`
- `rust/src/metrics.rs`
- `rust/src/health.rs`
- `rust/src/node.rs`
- `rust/src/lib.rs` (core types only)
- `rust/tests/replay_tests.rs`
- `rust/tests/blockchain_tests.rs`
- `rust/tests/integration_tests.rs`
- `rust/tests/crypto_tests.rs`
- `rust/benches/`
- `rust/Makefile` (integration gate)

**You do NOT touch:**
- `rust/src/solana_agent/` (Agent 2)
- `rust/src/proof/` (Agent 3)
- `web/` (Agent 2/3)
- `cpp/` (deprecated)

## Daily Duty

Every session starts with:
```bash
cd rust && make integration-check
```

If it fails, fix before any feature work.

## Canonical Rules

1. **No mock cryptography.** Use `ed25519-dalek` 2.0 only.
2. **No placeholder responses.** Every function returns real data or a typed error.
3. **No silent fallbacks.** Any fallback path logs at `WARN` minimum.
4. **No hidden autogenerated state.** All state changes are in `lib.rs` or `storage.rs`.
5. **SHA-256 everywhere.** No other hash algorithm in core.
6. **Deterministic execution.** Same inputs → same outputs. Tested in `replay_tests.rs`.
7. **libp2p 0.53+ only.** No duplicate networking layers.
8. **RocksDB only.** No duplicate storage abstractions.

## Integration Contract

When Agent 2 or 3 needs core services, they call the public API in `lib.rs`:

```rust
pub use crypto::{KeyPair, PublicKey, PrivateKey, DigitalSignature, CryptoError};
pub use storage::{BlockchainStorage, StorageError};
pub use network::{P2PNode, NetworkConfig, NetworkMessage};
pub use consensus::{HotStuffConsensus, ConsensusMessage, ConsensusAction, ConsensusError, Phase};
pub use metrics::{BlockchainMetrics, MetricsServer, Timer};
pub use health::{HealthChecker, HealthCheck, HealthStatus, HealthStatusResponse, AlertManager, Alert, AlertSeverity, SystemMetrics};
```

**Do not expose internals.** If an agent needs something not in this list, open a schema change request.

## Adding Dependencies

Agent 1 is the dependency gatekeeper. Any new crate:
1. Must pass `cargo audit`
2. Must not conflict with existing versions
3. Must be documented in `ARCHITECTURE.md`
4. Must have a one-sentence justification in the commit message

## Testing Requirement

Every change must include:
- Unit test in the module's `#[cfg(test)]` block
- Replay test if it affects state transitions
- Integration test if it affects cross-module behavior

No PR without tests.

## Forbidden Patterns

- `unsafe` blocks without `# SAFETY` comment and three-agent review
- `unwrap()` or `expect()` in production code (use `?` or typed errors)
- `println!` in library code (use `tracing`)
- `std::time::SystemTime::now()` in hash computation (breaks determinism)
- Floating-point in consensus or state logic
