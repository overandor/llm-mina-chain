# LLM-Mina-Chain: Agent Architecture & Boundaries

This document defines the three-agent split and the canonical contracts between them.

## Agents

### Agent 1 — Core Runtime (Authoritative)
**Directory:** `rust/src/` (crypto, storage, network, consensus, metrics, health, node)  
**Owner:** Core correctness, deterministic execution, compilation gate.

- Ed25519 signatures, SHA-256 hashing, key management
- libp2p 0.53+ gossipsub, mDNS, noise, yamux
- HotStuff BFT consensus (prepare / pre-commit / commit / decide)
- RocksDB persistent storage with batch writes
- Prometheus metrics + health checks
- Replay validation and deterministic state transitions
- **No UI, no product concepts, no Solana logic.**

### Agent 2 — Query + API (Data Plane)
**Directory:** `rust/src/solana_agent/` (RPC, query engine, REST, WebSocket)  
**Owner:** External data, analytics, API stability.

- Solana RPC client with failover
- SQL-like query engine over on-chain data
- Axum REST API with OpenAPI docs
- WebSocket streaming endpoints
- Dune-style query abstractions
- **No cryptography, no consensus, no proof generation.**

### Agent 3 — Proof + Provenance (Trust Plane)
**Directory:** `rust/src/proof/` (receipts, merkle chains, IPFS, anchoring)  
**Owner:** Trust infrastructure, deterministic proof state.

- Proof-of-Inference receipt generation
- Merkle chain state commitments
- Replay verification harness
- IPFS content addressing
- Solana anchoring (settle proof hashes on-chain)
- Audit logs with cryptographic integrity
- **No networking, no consensus, no query logic.**

| Hash Algorithm | SHA-256 everywhere | Agent 1 |

## Integration Gate

Every day, all agents merge to `integration` and run:

```bash
cargo test --workspace
cargo clippy -- -D warnings
cargo test --test replay_tests
cargo test --test integration_tests
```

If any step fails, stop feature work until green.

## Forbidden Patterns

- Mock cryptography in integration branches.
- Placeholder responses in integration branches.
- Duplicate networking layers (only libp2p in Agent 1).
- Duplicate storage abstractions (only RocksDB in Agent 1).
- Silent fallback behavior without `WARN` log.
- Hidden autogenerated state without audit trail.
